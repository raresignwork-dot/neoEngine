### 1. 頂点バッファと三角形の表示（最優先）

DirectX 11において、画面に「形」を出すための最小セットを実装します。

* **内容:** 3つの点の座標データ（頂点バッファ）を作り、GPUに送って三角形を描画します。
* **学ぶこと:** * `ID3D11Buffer` の使い方。
* 頂点シェーダー（VS）とピクセルシェーダー（PS）の役割。


* **クラス化への影響:** `Mesh` クラスや `Model` クラスを設計する際の基礎知識になります。

### 2. HLSL（シェーダー）ファイルの外部読み込み

コードの中にシェーダーを直接書くのではなく、
`.hlsl` ファイルとして「物理フォルダ」に保存し、
それを読み込んでコンパイルする仕組みを作ります。

* **内容:** `D3DCompileFromFile` 関数を使ってシェーダーをロードします。
* **学ぶこと:** * 外部ファイルの読み込みパスの指定方法。
* シェーダーのコンパイルエラーのデバッグ方法。


* **クラス化への影響:** `Shader` 管理クラスや、アセットローダーの設計に役立ちます。

### 3. 定数バッファ（Constant Buffer）によるデータの受け渡し

CPU側（C++）の変数を、GPU側（HLSL）へリアルタイムに送る仕組みを実装します。

* **内容:** 「経過時間」や「座標」を送り、三角形を動かしたり色を変えたりします。
* **学ぶこと:** `ID3D11Buffer` を `D3D11_BIND_CONSTANT_BUFFER` として作る方法。
* **クラス化への影響:** `Camera` クラス（行列を送る）や
`Light` クラスを作る際に必須の知識です。

### 4. 行列変換（ワールド・ビュー・プロジェクション）

3D（または2D）の座標計算に欠かせない「行列」を導入します。

* **内容:** 数学ライブラリ（DirectXMath）を使い、三角形を回転・移動・拡大させます。
* **学ぶこと:** 座標変換の概念。
* **クラス化への影響:** すべての「ゲームオブジェクト」が持つべき 
`Transform` クラスの設計基準になります。

### 5. テクスチャの表示（画像の貼り付け）

単色の三角形ではなく、画像（PNG等）を表示させます。

* **内容:** 画像ファイルを読み込み、ポリゴンに貼り付けます（サンプラーとテクスチャリソース）。
* **学ぶこと:** `ID3D11ShaderResourceView` の扱い。
* **クラス化への影響:** `Texture` クラスや `Sprite` クラスを作る際に必要になります。

この設計のメリット

    Loadの自動化: Update ループの中でフラグ管理をする必要がありません。
    ChangeScene が呼ばれた次のフレームで、新しいシーンの Load() が
    自動的に一度だけ実行されます。

    型安全な指定: manager.ChangeScene<TitleScene>(); と書くだけで、
    TitleScene.cpp/h を読み込んで実行できます。

    メモリのクリーンアップ: std::unique_ptr を使うことで、
    シーンが切り替わる際に古いシーンのメモリを自動で解放（Unload呼び出し含め）
    してくれます。

    

#include <memory>
#include "Scene.h"

class SceneManager {
public:
    // シーンを切り替える関数（テンプレート版）
    template <typename T>
    void ChangeScene() {
        // 次のシーンを予約
        m_nextScene = std::make_unique<T>();
    }

    void Update() {
        // もし次のシーンが予約されていたら、ここで入れ替える
        if (m_nextScene) {
            if (m_currentScene) m_currentScene->Unload();

            m_currentScene = std::move(m_nextScene);
            m_currentScene->Load(); // ここで一度だけ読み込みが走る！
        }

        if (m_currentScene) m_currentScene->Update();
    }

    void Draw() {
        if (m_currentScene) m_currentScene->Draw();
    }

private:
    std::unique_ptr<Scene> m_currentScene; // 現在のシーン
    std::unique_ptr<Scene> m_nextScene;    // 次のシーン（予約用）
};

## 1️⃣ パーティクルの本質（超平たく）

> **大量の小さな画像（点・板）を
> 毎フレームちょっとずつ動かすだけ**

これが本質です。

---

## 2️⃣ あなたの推測を分解すると

> 「ある領域内で
> ある位置に来たらこう動く
> ベクトル複数を用意して
> そこに画像を配置する」

これは実際には：

* **位置（position）**
* **速度（velocity = ベクトル）**
* **寿命（life）**
* **見た目（sprite）**

を持つ粒を更新しているだけです。

👉 **完全に正解方向**

---

## 3️⃣ 最小構成のパーティクル（数学ほぼゼロ）

### データ

```cpp
struct Particle {
    Vec2 pos;
    Vec2 vel;
    float life;
};
```

### 更新

```cpp
p.pos += p.vel * dt;
p.life -= dt;
```

これだけです。

* ベクトル = (x, y)
* 足し算だけ

---

## 4️⃣ 「ある領域に来たらこう動く」の正体

### ① 単純条件分岐

```cpp
if (p.pos.y < groundY) {
    p.vel.y = -p.vel.y * 0.5f; // 跳ね返る
}
```

---

### ② 擬似的な「流れ」

```cpp
Vec2 wind = Vec2(1.0f, 0.0f);
p.vel += wind * dt;
```

👉 **これだけで風っぽく見える**

---

### ③ ノイズを足す（超重要）

```cpp
p.vel.x += random(-0.1f, 0.1f);
p.vel.y += random(-0.1f, 0.1f);
```

👉 **ランダムが“自然さ”を作る**

# 🌳 完成度の高い Node クラス（ゲームエンジン向け）

以下は「実際のゲームエンジンで使えるレベル」の Node クラスです。

```cpp
#pragma once
#include <vector>
#include <string>
#include <memory>
#include <DirectXMath.h>

using namespace DirectX;

class Node
{
public:
    // ---- 基本情報 ----
    std::string name;

    // ローカル変換（親からの相対）
    XMMATRIX localTransform = XMMatrixIdentity();

    // グローバル変換（ワールド空間）
    XMMATRIX globalTransform = XMMatrixIdentity();

    // このノードに紐づくメッシュ
    std::vector<int> meshIndices;

    // 親子関係
    Node* parent = nullptr;
    std::vector<std::unique_ptr<Node>> children;

public:
    Node() = default;
    explicit Node(const std::string& name) : name(name) {}

    // ---- 階層更新 ----
    void UpdateGlobal(const XMMATRIX& parentMatrix)
    {
        globalTransform = localTransform * parentMatrix;

        for (auto& child : children)
            child->UpdateGlobal(globalTransform);
    }

    // ---- 子ノード追加 ----
    Node* AddChild(std::unique_ptr<Node> child)
    {
        child->parent = this;
        Node* ptr = child.get();
        children.push_back(std::move(child));
        return ptr;
    }

    // ---- ノード検索（名前） ----
    Node* Find(const std::string& target)
    {
        if (name == target)
            return this;

        for (auto& child : children)
        {
            if (auto* found = child->Find(target))
                return found;
        }
        return nullptr;
    }

    // ---- ノード検索（パス指定）----
    // 例: "Root/Armature/RightHand"
    Node* FindByPath(const std::string& path)
    {
        size_t pos = path.find('/');
        std::string head = (pos == std::string::npos) ? path : path.substr(0, pos);
        std::string tail = (pos == std::string::npos) ? ""   : path.substr(pos + 1);

        if (name != head)
            return nullptr;

        if (tail.empty())
            return this;

        for (auto& child : children)
        {
            if (auto* found = child->FindByPath(tail))
                return found;
        }
        return nullptr;
    }

    // ---- ローカル行列の更新（位置・回転・スケール）----
    void SetLocalTransform(const XMMATRIX& m)
    {
        localTransform = m;
    }

    // ---- グローバル行列を取得 ----
    XMMATRIX GetGlobalTransform() const
    {
        return globalTransform;
    }

    // ---- デバッグ用：階層を表示 ----
    void PrintHierarchy(int depth = 0) const
    {
        for (int i = 0; i < depth; i++) printf("  ");
        printf("%s\n", name.c_str());

        for (auto& child : children)
            child->PrintHierarchy(depth + 1);
    }
};
```

# 🌟 この Node クラスが「充実している」理由

## 1. **名前検索ができる**
glTF / Assimp の階層構造では「特定のボーン名を参照する」ことが必須です。

```cpp
Node* hand = model->rootNode->Find("RightHand");
```

これができるだけで、  
武器のアタッチ・エフェクトの追従・アニメーション制御が一気に楽になります。

---

## 2. **パス検索ができる**
階層が深いモデルでは、パス指定が便利です。

```cpp
Node* hand = model->rootNode->FindByPath("Root/Armature/RightHand");
```

Unity や Unreal と同じ感覚で扱えます。

---

## 3. **AddChild() で安全に階層構築できる**
Assimp や glTF ローダーでノードを構築するときに便利。

```cpp
auto child = std::make_unique<Node>("RightHand");
Node* hand = parent->AddChild(std::move(child));
```

---

## 4. **localTransform / globalTransform を完全サポート**
- localTransform … 親からの相対
- globalTransform … ワールド空間

Renderer は globalTransform を使うので、階層構造が正しく反映されます。

---

## 5. **階層デバッグが簡単**
```cpp
model->rootNode->PrintHierarchy();
```

これで glTF の階層をそのまま確認できます。
