# 0️⃣ Gitの構造（最重要理解）

Gitは3層構造です：

```
作業フォルダ（Working Directory）
        ↓ git add
ステージ（Index）
        ↓ git commit
履歴（Repository）
```

---

# 1️⃣ 初期設定（最初に一度だけ）

```bash
git config --global user.name "YourName"
git config --global user.email "your@email.com"
```

ローカルだけで使うならダミーでも可。

---

# 2️⃣ リポジトリ作成

## 既存フォルダで開始

```bash
git init
```

→ `.git` フォルダが作られる

---

# 3️⃣ 状態確認（最頻出コマンド）

```bash
git status
```

何が変更されたか確認。

---

# 4️⃣ 変更を記録する（超重要）

## ① 追跡開始

```bash
git add ファイル名
```

全部まとめて：

```bash
git add .
```

## ② 履歴に保存（セーブポイント）

```bash
git commit -m "変更内容の説明"
```

---

# 5️⃣ 履歴を見る

```bash
git log
```

短縮表示：

```bash
git log --oneline
```

ブランチ含めた可視化：

```bash
git log --oneline --graph --all
```

---

# 6️⃣ ブランチ操作

## 作成

```bash
git branch ブランチ名
```

## 作成＋切替（よく使う）

```bash
git switch -c ブランチ名
```

## 切替

```bash
git switch ブランチ名
```

確認：

```bash
git branch
```

---

# 7️⃣ ブランチ統合

```bash
git switch master
git merge feature_x
```

---

# 8️⃣ 変更の差分確認

```bash
git diff
```

ブランチ比較：

```bash
git diff master..Window
```

---

# 9️⃣ 元に戻す（重要）

## 作業中の変更を破棄

```bash
git restore ファイル名
```

全部破棄：

```bash
git restore .
```

---

## コミットに戻す（危険）

```bash
git reset --hard コミットID
```

※未コミット変更は消える

---

# 🔟 Revert（安全に取り消す）

履歴を残したまま打ち消す：

```bash
git revert コミットID
```

---

# 1️⃣1️⃣ 特定ファイルだけ復元

```bash
git restore --source=master -- ファイル名
```

---

# 1️⃣2️⃣ 追跡解除

```bash
git rm --cached ファイル名
```

---

# 1️⃣3️⃣ reflog（最強の復元コマンド）

```bash
git reflog
```

resetしてもここから戻せることが多い。

---

# 1️⃣4️⃣ リモート（ローカルのみなら不要）

ローカル専用なら：

✔ push 不要
✔ pull 不要

Gitは完全にローカルで動く。

# 🔎 「すべて保存」の意味

Visual Studio の保存は：

```
メモリ → ファイルに書き出す
```

だけです。

Gitは：

```
ファイル → コミット（履歴に記録）
```

です。

つまり

| 操作         | 何をしているか |
| ---------- | ------- |
| Ctrl+S     | ファイル保存  |
| すべて保存      | 全ファイル保存 |
| git add    | 追跡開始    |
| git commit | 履歴に確定保存 |

# 🔥 今回なぜ消えたか

あなたは：

1. ブランチ作成
2. 修正
3. 保存（VS）
4. コミットせずにブランチ操作

`reset --hard` は特に危険で：

# 🎯 基本原則（最重要）

## ① commit = セーブポイント

作業単位ごとに必ず：

```bash
git add .
git commit -m "WIP: short description"
```

WIPコミットは悪ではありません。
個人開発では「安全」が最優先です。

---

## ② reset --hard を安易に使わない

危険コマンド：

```bash
git reset --hard
git clean -fd
```

使う前に必ず：

```bash
git branch backup_before_reset
```

を作る。

---

## ③ 実験は必ずブランチで行う

```bash
git switch -c feature/fullscreen
```

作業完了後：

```bash
git switch master
git merge feature/fullscreen
```

---

# 🏗 個人ゲーム開発向けブランチ戦略（シンプル版）

## main（または master）

常に「ビルド可能状態」

## develop（任意）

統合テスト用

## feature/*

機能単位ブランチ

例：

```
feature/renderer-refactor
feature/window-system
feature/scene-context-cleanup
```

---

# 🔒 安全作業フロー（推奨）

### 1. 作業開始

```bash
git switch master
git pull   (※リモート使う場合)
git switch -c feature/xxx
```

### 2. 作業中

こまめに：

```bash
git add .
git commit -m "WIP"
```

### 3. 完了後

```bash
git switch master
git merge feature/xxx
git branch -d feature/xxx
```
