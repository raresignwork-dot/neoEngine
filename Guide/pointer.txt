書き方           意味                        使える場面
player.hp       playerそのものが持つhp       Player型の変数に対して
player->hp      playerが指しているもののhp   Player*型のポインタに対して
(*player).hp    ↑と全く同じ（古風な書き方）   Player*型のポインタに対して

アドレス渡しが*で実態渡しが&。実態へのアクセスが.でポインタへの参照が->

Player p;              // スタックに実体を作る
Player* player = &p;   // playerはpの実体のアドレスを保持
player->hp = 100;      // OK
(*player).hp = 100;    // これもOK（全く同じ）

Player& p = new Player();   // ← コンパイルエラー（ほぼ確実に通らない）  
Player* p  = new Player();     // OK（ポインタ）
Player& r = *new Player();     // これも一応書けるが非常に危険（後述）
Player& player;             // ← これ単体ではほぼ意味がない（危険なコード）
Player& player;          // エラー：参照は初期化が必要
Player& player = ???;    // ← 何かしら実体を必ず束縛しないといけない

初期化例（正しい書き方）
Player obj;
Player& player = obj;          // OK：既存のオブジェクトへの参照

// または（newを使う場合の珍しい書き方）
Player& player = *new Player();   // 一応コンパイルは通るが…非常に危険！

解説
player が Player&（参照）型の場合 → これは正しく動きます
参照は「別名」のようなものなので、. で普通にメンバにアクセスできます。

Player obj;
Player& player = obj;
player.hp = 100;       // → obj.hp = 100 と同じ意味（完全に正しい）

解説
-> は ポインタ（Player*）に対して使う演算子です。
player が Player&（参照）の場合は コンパイルエラー になります。

Player* ptr = new Player();
ptr->hp = 100;         // OK

Player& ref = *ptr;
ref->hp = 100;         // エラー！ 参照には -> を使えない


解説
これが書けるのは、player が ポインタ（Player*）型の場合だけです。
(*player).hp = 100;    // 

Player* player = new Player();
(*player).hp = 100;       // OK（-> と同じ意味）
player->hp = 100;         // ほぼ同じ（現代ではこっちが主流）

スマートポインタ（std::unique_ptr、std::shared_ptr、std::weak_ptr）は、
基本的に生のポインタ（raw pointer）の * と & をほとんど使わずに済むように設計された型です。
これが現代C++（C++11以降）の大きな目標の一つです。

スマートポインタが「* と & を避けられる」主な理由と実際の使い方

操作                      生ポインタの場合       スマートポインタの場合          コメント
オブジェクトにアクセス     *ptr または ptr->      ptr-> または *ptr（両方OK）    生ポインタと同じ記法でOK
アドレスを取得             &obj や ptr 自体      ptr.get()                     .get() で raw pointer 取得（非所有）
所有権を渡す              new / delete 手動管理  std::make_unique / 
                                               std::make_shared + ムーブ      * も & も出てこない
関数に「見るだけ」渡す     const T* や const T&  const T& obj = *smart_ptr; 
                                               または smart_ptr.get()          ほとんど * 不要
メンバ変数として持つ      T* ptr;（所有権不明瞭） std::unique_ptr<T> や
                                               std::shared_ptr<T>             所有権が明確、delete不要

実際のコード例（* と & を極力使わないパターン）cpp

// 生ポインタ風（古いスタイル）
void OldStyle()
{
    Widget* w = new Widget(42);
    UseWidget(w);           // & も * も出てくる可能性大
    delete w;
}

// 現代スタイル（ほとんど * & が出てこない）
void ModernStyle()
{
    auto w = std::make_unique<Widget>(42);   // new も & もなし

    UseWidget(*w);          // ここだけ * が出る（ほぼこれだけ）
    // または
    UseWidgetByPtr(w.get()); // get() で raw pointer 渡し（非所有）

    // 関数終了で自動 delete → メモリリークなし
}

// 共有する場合
auto shared_w = std::make_shared<Widget>(100);
AnotherClass ac{shared_w};   // コピーするだけ（参照カウント自動管理）

例外的に * や & が出てくる場面（少ない）関数に「参照で渡したい」けどスマートポインタを渡せないとき
→ UseWidget(*my_unique_ptr);  ← これが一番よく出る *
CスタイルAPIに渡すとき（DirectXなど）
→ device->CreateBuffer(..., my_buffer.get(), ...);  ← get() を使う
意図的に生ポインタが必要な低レベルコード
→ ただし、これは最小限に留めるのが現代の流儀

特に std::make_unique / std::make_shared と組み合わせると、

現代C++の理想は「生ポインタは所有権を表さない非所有参照としてだけ使う」なので、
スマートポインタがメインになると * と & の出番は本当に激減します。
